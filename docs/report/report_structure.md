# Report Structure

This document describes the structure of the JSON report generated by the `codesage report` command.

## Project Summary

The `project` object contains a summary of the project's health.

- `total_files`: The total number of files in the project.
- `high_risk_files`: The number of high-risk files.
- `medium_risk_files`: The number of medium-risk files.
- `low_risk_files`: The number of low-risk files.
- `total_issues`: The total number of issues.
- `error_issues`: The number of error-level issues.
- `warning_issues`: The number of warning-level issues.
- `info_issues`: The number of info-level issues.
- `top_rules`: A list of the top issue rules.
- `top_risky_files`: A list of the top risky files.

## File Summary

The `files` array contains a summary of each file's health.

- `path`: The path to the file.
- `risk_level`: The risk level of the file.
- `risk_score`: The risk score of the file.
- `loc`: The number of lines of code in the file.
- `num_functions`: The number of functions in the file.
- `issues_total`: The total number of issues in the file.
- `issues_error`: The number of error-level issues in the file.
- `issues_warning`: The number of warning-level issues in the file.
- `top_issue_rules`: A list of the top issue rules in the file.
